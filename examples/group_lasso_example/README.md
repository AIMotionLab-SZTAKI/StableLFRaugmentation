# Model Order Selection Example

## Usage
To reproduce the experiments, navigate to the directory containing the benchmark scripts and display the available command-line options:
```bash
cd StableLFRaugmentation/examples/group_lasso_example/
python3 gl_example.py -h 
```

This command returns the following description of the script and its arguments:
```text
usage: gl_example.py [-h] [--seed SEED] [--nxa NXA] [--nwa NWA] [--nza NZA] [--var {x,w,z}] [--rho RHO]

Example script for testing the group-lasso regularization option for LFR-based model augmentation.

options:
  -h, --help     show this help message and exit
  --seed SEED    Random seed.
  --nxa NXA      Dimension of the augmented state. If >0 dynamic augmentation, if ==0 static augmentation.
  --nwa NWA      Dimension of the latent variable w_a.
  --nza NZA      Dimension of the latent variable z_a.
  --var {x,w,z}  Select the variable on which the group-lasso regularization is applied.
  --rho RHO      Group-lasso regularization coefficient.
```

The file `hyperparams.yaml` contains the hyperparameter configuration used to generate the results reported in the paper.
Modifying this file allows users to explore alternative configurations; however, such changes naturally can lead to different model performance.

> [!WARNING]
> Note that the reported results in the paper were obtained using a slightly modified implementation.
> Consequently, the results generated by this example script may not be the same, although the deviation is expected to be minimal.

## Example
To reproduce the results discussed in the paper, it is enough to vary two parameters, namely `var` and `rho`.
For instance, to conduct model training with group-lasso regularization on the augmented states $x_\mathrm{a}$ with coefficient $\rho_\mathrm{x_a}=10^{-3}$, run the following:
```bash
python3 gl_example.py --var x --rho 1e-3
```

After model training, the script also conducts a sparsity analysis on the model parameters and identifies redundant dimensions. In this specific example, the script outputs:
```text
Sparsity analysis results:
xa_1 can be eliminated
xa_2 can be eliminated
```

The same process can be repeated with multiple regularization coefficient values for $x_\mathrm{a}$, $w_\mathrm{a}$, and $z_\mathrm{a}$, respectively.
